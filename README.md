# QRCommunication

画面に表示した QR コードをカメラで連続スキャンして、テキスト・ファイル・
GitHub リポジトリを機器間で転送するブラウザアプリです。サーバーを介さず、
2 台のブラウザ（送信側・受信側）だけでデータをやり取りします。

## 特徴

- **サーバーレス**: バックエンドなし。送信側は QR コードを連続表示し、
  受信側はカメラでそれを読み取るだけでデータを復元します。
- **3 つの送信モード**: テキスト／ファイル／GitHub リポジトリ
  （`owner/repo[@ref]` や URL を指定すると `api.github.com` から
  zipball を取得して送信、公開リポジトリのみ対応）。
- **範囲指定再送**: 受信側で欠損したチャンク番号だけを送信側に伝えて
  再送できます。
- **Wake Lock 対応**: QR 送信中にモバイル端末がスリープしないよう
  画面ロックを抑止します。
- **設定を保存**: チャンクサイズ・FPS・誤り訂正レベル・QR バージョン・
  セルサイズ・カメラ解像度などを細かく調整でき、`localStorage` に保存
  されます。外部への送信は GitHub リポジトリ取得時のみです。

### 未受信チャンク番号の伝達（QR / DTMF）

ネットワークで繋がっていない2台間で未受信チャンク番号をやり取りする方法を
2つ用意しています。

- **QRブリッジ（📷 QRで受け取る／📱 QRで送る、推奨）**: 受信側が未受信範囲を
  小さなQRとして表示し、送信側がそれを一度だけカメラで読み取って範囲指定欄に
  自動入力します。メインの送受信ループとは別のカメラ／キャンバスを使うため、
  やり取りしている間も両端末のセッション（送信ループ・受信スキャン）は
  途切れません。
- **DTMF音（🎤 音で受け取る／🔊 音で送る）**: 音声トーンでのやり取りです。
  実際の利用環境ではうまく認識できないことが多く、実験的な機能という
  位置づけです。

どちらもうまくいかない場合は、未受信のチャンク番号を目視で確認し、
範囲指定欄に手入力して再送してください。

## 使い方

1. **送信側**: 「送信」タブでテキスト／ファイル／GitHub リポジトリを選び、
   「QR表示開始」を押します。QR コードが連続的に切り替わって表示されます。
2. **受信側**: 「受信」タブで「カメラ開始」を押し、送信側の画面を映します。
   全チャンクを受信するとテキストのコピーやファイルのダウンロードができます。
3. 送信中に画面を離れても、QR は最初から繰り返し表示されるため、途中から
   スキャンを始めても問題ありません。
4. 受信で欠けたチャンクがある場合は、未受信番号を確認し、送信側の
   「範囲指定」欄にその範囲を入力して「反映」を押すと、該当分だけ
   再送されます。
5. 「設定」タブでチャンクサイズ・FPS・誤り訂正レベルなどを調整すると、
   転送速度や読み取り精度を環境に合わせて最適化できます。

## 動作要件

カメラ（`getUserMedia`）を使うため、`https://` または `http://localhost`
での配信が必要です。それ以外の URL ではカメラ起動に失敗する可能性が
あります。

## 使用ライブラリ

`vendor/` 以下にバンドルしています。ビルドは不要で、`index.html` を
直接開くだけで動作します。
